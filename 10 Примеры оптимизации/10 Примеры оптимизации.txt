------------------------------ ER-МОДЕЛЬ:
Entity-Relationship model		-модель «сущность — связь». она определяет сущности и связи между сущностями.
					-модель данных, позволяющая описывать концептуальные схемы предметной области. 
ER-модель — Википедия			https://ru.wikipedia.org/wiki/ER-%D0%BC%D0%BE%D0%B4%D0%B5%D0%BB%D1%8C
Сущность (entity) 			-это реальный или представляемый тип объекта, информация о котором должна сохраняться и быть доступна.
Связь (relationship) 			-это графически изображаемая ассоциация, устанавливаемая между двумя сущностями. Связь может существовать между двумя разными сущностями или между сущностью и ей же самой (рекурсивная связь). Связь обычно задается ограничением в виде FOREIGN KEY. https://www.postgresql.org/docs/14/tutorial-fk.html Возможны связи на основе отношений:
	один-к-одному 			-на 1-запись первой таблицы соответствует 1-запись второй  	(не используется практически)
	один-ко-многим			-на 1-первой - много во второй 					(справочник)
	многие-ко-многим 		-промежуточная таблица, совмещающая варианты соединений		(авторы и книги)


============================== 102 СЕКЦИОНИРОВАНИЕ / PARTITIONING:
разбиение одной большой логической таблицы на несколько меньших физических секций, размер таблицы не увеличивется, а скорость работы запроса вырастает:
	-Уменьшается выборка при последовательном cканировании (sequential scan);
	-Индексы меньше.
PARTITION BY { RANGE | LIST | HASH } ( { имя_столбца | ( выражение ) } [ COLLATE правило_сортировки ] 	*-сделать разделение таблицы по RANGE | LIST | HASH
[ класс_операторов ] [, ... ] ) 
CREATE TABLE section_name PARTITION OF table_name FOR VALUES FROM ('2006-02-01') TO ('2006-03-01');	*-создать таблицу для PARTITION с диапозоном в один месяц. При вставлении данных в таблицу, данные будут попадать в ту или иную секцию "по-месячно".
DROP TABLE section_name											+позволяет удалять партиции/секции с данными.
------------------------------ Ценность секционирования:
1/	Увеличивает быстродействие/производительности, особенно когда большой процент часто запрашиваемых строк таблицы относится к одной или лишь нескольким секциям;
2/	Массовую загрузку и удаление данных можно осуществлять, добавляя и удаляя секции, если такой вариант использования был предусмотрен при проектировании секций;
3/	Удаление отдельной секции командой DROP TABLE и действие ALTER TABLE DETACH PARTITION выполняются гораздо быстрее;
4/	Когда в выборке или изменении данных задействована большая часть одной секции, производительность может возрасти, если будет выполняться последовательное сканирование (sequential scan) этой секции, а не поиск по индексу (index scan), сопровождаемый произвольным чтением данных, разбросанных по всей таблице;
5/	Редко используемые данные можно перенести на более дешёвые и медленные носители;
------------------------------ Виды секционирования:
Секционирование по диапазонам 		-по ключевому столбцу или набору столбцов, и не пересекающимся друг с другом;
Секционирование по списку 		-с помощью списка, явно указывающего, какие значения ключа должны относиться к каждой секции;
Секционирование по хешу 		-по определённым модулям и остаткам, которые указываются для каждой секции.
------------------------------ СЕКЦИОНИРОВАНИЕ ПО ДИАПАЗОНУ ЗНАЧЕНИЙ:
CREATE TABLE table_2020_02 PARTITION OF table FOR VALUES FROM (‘2020-02-01’) TO (‘2020-03-01’)		-по полю "table", будут попадать данные за февраль в таблицу table_2020_02
CREATE TABLE table_2020_03 PARTITION OF table FOR VALUES FROM (‘2020-03-01’) TO (‘2020-04-01’)		-по полю "table", будут попадать данные за март в таблицу table_2020_03
CREATE TABLE table_2020_04 PARTITION OF table FOR VALUES FROM (‘2020-04-01’) TO (‘2020-05-01’)		-по полю "table", будут попадать данные за апрель в таблицу table_2020_04
	FROM (‘2020-03-01’)					--включая
	TO (‘2020-05-01’)					--до значения

------------------------------ СЕКЦИОНИРОВАНИЕ ПО СПИСКУ HASH:
*-Когда необходимо равномерное распределение;
*-Параллелизация операций;
*-Нет явного ключа, по которому можно разбить таблицу;
*-Для равномерного распределения необходимо уникальное или почти уникальное поле.
CREATE TABLE table_p1 PARTITION OF table FOR VALUES WITH (MODULUS 5, REMAINDER 0);			-по полю "id". "MODULUS 5"-создаст 5-пять секций



============================== 103 СТЕ - Common Table Expression:
Обощённое табличное выражение	 	— это временный результат запроса, который можно использовать в основном SQL-запросе. 
Общее табличное выражение	 	— это именованный временный набор данных, определяемый с помощью ключевого слова WITH. 
СТЕ					- именованный подзапрос в запросе.
------------------------------ Преимущества CTE:
*-Упрощение кода — CTE позволяет разбивать сложные SQL-запросы на более понятные части. Для снижения сложности запроса;
*-Повторное использование данных — можно обращаться к CTE несколько раз в основном запросе, избегая дублирования кода;
*-Ускорение работы с рекурсией — CTE позволяет выполнять рекурсивные запросы, что полезно, например, для обработки иерархических данных (например, древовидных структур).
WITH temporary_table 						-
AS 
(
 SELECT column1, column2 FROM some_table WHERE condition	-подзапрос
 ) 	-
SELECT * 
FROM temporary_table; 						-
------------------------------ Задачи CTE:
1/	WITH предоставляет способ записывать дополнительные операторы для применения в больших запросах;
2/	можно делать несколько СТЕ и из последующего вызывать предыдущее;
3/	значительно упрощает чтение/восприятие синтаксиса запроса;
4/	можно представить как определения временных таблиц, существующих только для одного запроса;
5/	они вычисляются только раз для всего родительского запроса, даже если этот запрос или соседние запросы WITH обращаются к ним неоднократно;
6/	есть возможность не материализовывать (использовать временную таблицу - долго) https://habr.com/ru/post/440576/ 
*-Таким образом, сложные вычисления, результаты которых нужны в нескольких местах, можно выносить в запросы WITH в целях оптимизации
