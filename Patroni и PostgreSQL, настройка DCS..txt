=================== Настройка и администрирование кластеров Patroni:
это процесс создания, управления и поддержки высокодоступного кластера СУБД PostgreSQL, который автоматически обеспечивает отказоустойчивость и переключение на резервный узел (failover/switchover) при сбое, используя репликацию и распределенное хранилище конфигурации (DCS), такое как etcd, Consul, ZooKeeper, Kubernetes. 
Patroni 				.это автоматическая система аварийного переключения для PostgreSQL. 
Patroni 				.обеспечивает автоматическое и ручное переключение при отказе и хранит все важные данные в распределенном хранилище конфигурации (DCS) на базе систем ETCD, Consul и т.д.
Patroni 				.автоматизирует сложные задачи управления кластером, позволяя легко масштабировать его и поддерживать консистентность данных. Основные задачи при настройке/администрировании:
Развертывание: 				.Установка Patroni и PostgreSQL на нескольких узлах, настройка DCS.
Конфигурация: 				.Определение параметров кластера, репликации и DCS в конфигурационных файлах Patroni, например, для https://patroni.readthedocs.io/.
Автоматизация Failover/Switchover: 	.Patroni мониторит состояние узлов и при сбое основного (master) автоматически выбирает нового лидера, перенаправляя трафик.
Мониторинг: 				.Отслеживание состояния узлов, реплик и DCS для обеспечения здоровья кластера.
Управление репликацией: 		.Добавление новых реплик, изменение их типа (синхронная/асинхронная).
Интеграция с балансировщиками: 		.Настройка HAProxy или PgBouncer для маршрутизации запросов к активному узлу.
Динамическое конфигурирование: 		.Возможность изменять настройки PostgreSQL без перезапуска всего кластера. 
В итоге, это комплекс действий по созданию надежной и отказоустойчивой базы данных PostgreSQL с минимальным участием администратора для рутинных операций. 
==================== DCS в контексте PostgreSQL:
Distributed Configuration Store 	.распределенное хранилище конфигурации. Это внешняя система (например, etcd, Consul, ZooKeeper), которая используется фреймворками высокой доступности (типа Patroni) для координации узлов кластера, хранения их состояния, обеспечения консенсусуса и автоматического переключения при отказах, чтобы PostgreSQL кластер оставался доступным. 
Distributed Configuration Store		.Это надежное хранилище «ключ-значение», которое гарантирует, что все узлы кластера знают актуальный статус друг друга и могут выбрать лидера (мастера) без человеческого вмешательства.
